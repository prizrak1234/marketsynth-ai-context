# Architecture Core

Status: ACTIVE

## 1. System Shape

Marketsynth is structured as a governed AI runtime with specialist domains.

Core layers:

1. Product Constitution
2. Architecture Core
3. Runtime Invariants
4. Domain Runtime
5. Contracts
6. Execution Layer
7. Evidence and Audit Layer
8. Knowledge Candidate Layer

## 2. Core Principle

The system is not a chain of prompts.

It is an auditable runtime where every significant state transition is traceable.

## 3. Domain Model

Initial domains:

- Marketing
- Programmer
- Automation
- Knowledge
- Execution
- Orchestration

Future domains:

- HR
- Legal
- Sales
- Finance
- Operations

## 4. Execution Model

Execution must pass:

1. Context
2. Task
3. Candidate output
4. Quality check
5. Readiness gate
6. Human approval
7. Real execution
8. Evidence capture
9. Outcome registration
10. Knowledge candidate evaluation

## 5. Separation of Concerns

Architecture must separate:

- reasoning from execution;
- draft from approved artifact;
- readiness from approval;
- evidence from interpretation;
- tenant-local knowledge from shared product logic.

## 6. Implementation Rule

Code must adapt to architecture when feasible.

Architecture may be refined only when audit proves that implementation reality exposes a valid flaw.
