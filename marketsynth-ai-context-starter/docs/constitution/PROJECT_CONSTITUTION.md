# Project Constitution

Status: FROZEN  
Project: Marketsynth

## 1. Mission

Marketsynth is an AI-assisted operational platform for marketing, automation and future specialist domains.

The system must be built as a governed runtime, not as a loose prompt chain.

## 2. Source of Truth

Architecture and approved governance documents define the project truth.

Chat history, temporary notes, generated drafts and local experiments are not authoritative unless promoted into this repository.

## 3. Human Control

Human approval is mandatory before real-world execution.

This includes:

- publication to external channels;
- spending or budget changes;
- modifying production data;
- external account actions;
- irreversible changes;
- credentials or secret handling.

## 4. Domain Expansion

The architecture must support adding future domains such as HR, Legal, Sales, Finance or Operations without global rewrite.

## 5. Tenant Isolation

Tenant isolation is a constitutional invariant.

No runtime memory, knowledge candidate, evidence, decision record or execution artifact may cross tenant boundaries.

## 6. Engineering Discipline

Implementation must follow:

- typed contracts;
- explicit state transitions;
- auditability;
- deterministic identifiers where needed;
- safe error handling;
- cost monitoring;
- rollback strategy;
- test coverage.

## 7. AI Agent Limits

AI agents may assist, generate drafts, inspect code and suggest changes.

AI agents must not silently override frozen architecture, change security boundaries or bypass approval gates.
