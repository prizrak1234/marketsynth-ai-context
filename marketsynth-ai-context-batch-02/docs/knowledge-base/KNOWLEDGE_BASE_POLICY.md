# Knowledge Base Policy

Status: ACTIVE

## Definition

The Knowledge Base is curated engineering knowledge for AI agents.

It is not a dump of chats, drafts or experiments.

## Allowed Knowledge

- product knowledge;
- architecture knowledge;
- engineering rules;
- domain concepts;
- operational runtime rules;
- accepted decisions;
- validated lessons learned.

## Not Authoritative

Cursor must not treat these as truth:

- raw chat logs;
- unreviewed ideas;
- draft RFCs;
- screenshots without extracted decisions;
- old abandoned names;
- generated code without architectural validation.

## Promotion Rule

Knowledge becomes authoritative only when:

1. written into this repository;
2. assigned a status;
3. scoped clearly;
4. compatible with Constitution;
5. compatible with Runtime Invariants.

## Tenant Rule

Knowledge Candidate never crosses tenant boundary.
